# critique.md — 5+1 維度專家評審

四個 skill 共用的產出評審機制。**評產出，不評人。**

**何時跑**：對方提「評審」「打分」「這樣好不好」「review 一下」；或你交付前對產出沒把握、想主動質檢時提議跑一次。**不是每次交付都跑**——這是深度質檢，不是例行步驟（例行的是顏色自審＋渲染驗證＋craft 四題）。

**評什麼**：HTML 視覺產出（prototype／report／BRD／PRD HTML 版）與 demo 影片（抽幀評）。PRD md 版是給 AI 開發的內容文件，不適用本檔（那要評的是邊界條件窮舉度，屬內容評審）。

**怎麼評**：對著 `verify.py` 的截圖（影片對著抽幀）逐維度打分——評分數的依據必須指得出畫面上的具體位置，不打印象分。

> 蒸餾自 [huashu-design](https://github.com/alchaincyf/huashu-design)（alchaincyf，MIT License）的 critique-guide，「哲學一致性」維度依我們的 CI 前提改為「品牌一致性」，場景側重表改為本工具箱的產出類型。

## 維度 0・概念/立意（權重最高，一票否決）

先問「這份產出有沒有一個 idea」，再看做得好不好——執行是放大器，放大一個空洞的概念只會更空洞。

| 分數 | 標準 |
|---|---|
| 9-10 | 有一個從**這次內容**長出來的獨有視覺主張，母題不可替換 |
| 7-8 | 有明確立意，母題與內容相關但換個近似功能也勉強能用 |
| 5-6 | 只有整潔沒有概念：乾淨，但沒說任何東西 |
| 3-4 | 通用模板套皮，概念層為零 |
| 1-2 | 連基本取捨都沒有，純元素堆疊 |

核心問題清單：
- 這份產出說了什麼？能用一句話講出它的 idea 嗎？講不出來就沒有
- **換個功能名/公司名還成立嗎？成立 = 模板，本維度直接 ≤5 分**（與 html-craft §10 第 1 題同源，但這裡是打分不是自問）
- 版面重心跟這次內容的重心是同一件事嗎？（排序結論的提案，排序表是不是視覺重心？）

**一票否決**：概念 ≤5 分時，總評封頂 6.0。先修概念再談執行——把模板打磨得更亮不會變成設計。

## 維度 1・品牌一致性

| 分數 | 標準 |
|---|---|
| 9-10 | 蒙住 logo 仍看得出是這間公司的東西：CI 全溯源、brand-voice 的語氣與設計模式落實到細節 |
| 7-8 | CI 正確、語氣對，但少數細節仍是「通用好看」而非「這間公司的好看」 |
| 5-6 | 顏色對了但氣質不對——套了 tokens 的通用模板 |
| 3-4 | 有 CI 外的顏色/字型，或語氣與 brand-voice 明顯不符 |
| 1-2 | 與公司 CI 基本無關 |

評審要點：全部 hex 溯源 tokens？禁用舊色有沒有出現？語氣符合 brand-voice？accent 用在品牌該強調的地方？

## 維度 2・視覺層級

| 分數 | 標準 |
|---|---|
| 9-10 | 視線自然沿設計意圖流動，資訊獲取零摩擦 |
| 7-8 | 主次清晰，偶有 1-2 處層級模糊 |
| 5-6 | 分得出標題和正文，但中間層級混亂 |
| 3-4 | 資訊平鋪，沒有明確視覺入口 |
| 1-2 | 混亂，不知道先看哪裡 |

評審要點：標題與正文字號對比 ≥2.5 倍？字重/大小/顏色建立了 3-4 個清晰層級？瞇眼測試——瞇起眼看層級仍清晰？留白在引導視線還是只是空著？

## 維度 3・細節執行

| 分數 | 標準 |
|---|---|
| 9-10 | 對齊、間距、密度節奏無可挑剔 |
| 7-8 | 整體精緻，1-2 處微小對齊/間距問題 |
| 5-6 | 基本對齊，但間距不成系統（不是三個數量級，是隨手填的） |
| 3-4 | 明顯的對齊錯誤、間距混亂、破版 |
| 1-2 | 粗糙，像草稿 |

評審要點：間距是否分三個數量級（html-craft §3）？同類元素間距一致？對齊方式全局統一？中文排印三件事（fallback 鏈/無斜體/引號）有沒有做對？

## 維度 4・功能性

| 分數 | 標準 |
|---|---|
| 9-10 | 每個元素都服務於這次的溝通目的，零冗餘 |
| 7-8 | 目的明確，有少量可刪的裝飾 |
| 5-6 | 基本可用，但有明顯分散注意力的裝飾元素 |
| 3-4 | 形式大於功能，找關鍵資訊要費力 |
| 1-2 | 被裝飾淹沒，失去傳達能力 |

評審要點：刪掉任何一個元素版面會變差嗎（不會就該刪）？受眾最在意的資訊在最顯眼的位置？prototype 的關鍵交互找得到、點得動？假數據裝飾（沒標示意值的編造數字）出現即扣。

## 維度 5・創新性

| 分數 | 標準 |
|---|---|
| 9-10 | 在 CI 框架內找到讓人記得住的表達 |
| 7-8 | 有自己的想法，不是模板套用 |
| 5-6 | 中規中矩 |
| 3-4 | 大量 AI slop cliché（html-craft §5 黑名單命中） |
| 1-2 | 純模板拼湊 |

## 場景側重（概念維不參與取捨——它對所有場景都是第一道關）

| 產出 | 最重要 | 次重要 | 可放寬 |
|---|---|---|---|
| prototype（demo 級） | 功能性、細節執行 | 視覺層級 | 創新性（可用性優先） |
| report（對 CEO/主管） | 視覺層級、概念表達力 | 品牌一致性 | 創新性（清晰優先） |
| BRD／PRD HTML | 功能性、細節執行 | 視覺層級 | 創新性（專業優先） |
| demo 影片（抽幀評） | 視覺層級、節奏（每段停留夠不夠） | 品牌一致性 | 細節執行（動態中看不清的微對齊） |

## 評審輸出模板

```
## 產出評審：[檔名]

**總體評分**：X.X/10 [優秀(8+)/良好(6-7.9)/需改進(4-5.9)/不合格(<4)]
（概念 ≤5 時總評封頂 6.0，先修概念再談執行）

**分項**：
- 概念/立意：X/10 [這份產出的 idea 是什麼？一句話]
- 品牌一致性：X/10 [一句話]
- 視覺層級：X/10 [一句話]
- 細節執行：X/10 [一句話]
- 功能性：X/10 [一句話]
- 創新性：X/10 [一句話]

### Keep（做得好的，具體指出位置）
- …

### Fix（按嚴重程度排序）
**1. [問題名]** — ⚠️致命 / ⚡重要 / 💡優化
- 當前：[截圖上的現狀]
- 問題：[為什麼是問題]
- 修復：[具體操作，含數值——「間距改 48px」不是「間距調大一點」]

### Quick Wins（只有 5 分鐘就先做這 3 件）
- [ ] …
- [ ] …
- [ ] …
```

評完問對方：「要我照 Fix 清單修嗎？還是只修 Quick Wins？」——評審不自動觸發修改。
