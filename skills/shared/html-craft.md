# html-craft.md — HTML 產出的工藝守則

`pm-propose`（BRD／PRD HTML／report）與 `pm-prototype` 產 HTML 時共用的 craft 層。CI 嚴格模式管的是「不用錯顏色」；這份管的是「不做出平庸的版面」——兩者都要遵守，衝突時 CI 嚴格模式優先（顏色永遠只來自 `brand-tokens.css`）。

> 方法論蒸餾自 [huashu-design](https://github.com/alchaincyf/huashu-design)（alchaincyf，MIT License）的 typography／content-guidelines／反 AI slop 體系，依 pm-almighty-box 的定位（企業內部 PM 文件與 prototype）改寫。

## 0. 為什麼需要這份

AI 沒有具體規則時，輸出的是訓練語料的視覺平均值——均一的圓角卡片、均一的間距、兩檔字重、每個標題配 icon。這種產出可以 100% 符合 CI 卻依然看起來像「又一個 AI 做的頁面」。這份守則的目的：把版面品質的下限從「AI 平均值」抬到「有人設計過」。

## 1. 硬底線（任何產出不得違反）

- 正文 ≥ 14px；標籤／註釋 ≥ 12px
- 正文對比度 ≥ 4.5:1（淺灰字別淡到讀不見；muted 文字只用於輔助資訊，不用於關鍵內容）
- 可點擊元素 hit target ≥ 44px（prototype 的按鈕尤其）
- 留白必須是**構圖**，不是內容缺席——每屏／每頁要有明確的視覺錨點，視線有落點。大片死白＋內容擠在角落 = 破版，不是極簡

## 2. 字重與排印

- **至少用三檔字重做層次**（例如 400／500／700），不要只靠字級大小區分階層。大標題可以降字重、加大字級（大而輕比大而粗更有質感）
- 大標題（≥28px）加負 letter-spacing（-0.5px ~ -1.5px）；小型全大寫英文標籤加正 letter-spacing（+0.5px ~ +4px）
- 行高與行長聯動：正文行高 1.6-1.7、每行 ≤ 38 個全形字；密集表格行高可到 1.4
- 中文排印三件事：字型 fallback 鏈西文在前中文在後（英數用西文字型渲染才不會歪）；**中文沒有斜體**——強調改用字重、accent 色或底線；引號用「」不用 ""
- 字型只用 `brand-tokens.css` 定義的變數，不外連任何 webfont

## 3. 空間層次：三個數量級

間距系統要有明顯的層級差，讓結構不靠框線也讀得出來：

| 層級 | 用途 | 參考值 |
|---|---|---|
| 大 | 頁邊距、章節之間 | 48-96px |
| 中 | 區塊／卡片之間 | 24-40px |
| 小 | 元素內部（標題與內文、label 與值） | 4-12px |

全頁間距均一（例如什麼都是 16px）= 沒有結構。相鄰資訊靠近、不相關資訊拉開，分組靠空間不靠加框。

## 4. 單一 accent 紀律

- 中性色（黑／灰／白／淺底）承擔 90% 的版面，**品牌主色當 accent，只出現在刻意選定的少數位置**：關鍵數字、當前狀態、主要 CTA、章節標記
- 主色滿版鋪開（每個卡片都描橘邊、每個標題都橘色）會讓它失去強調功能——到處都重點＝沒有重點
- 需要「稍淺／稍深」的變化用既有 token 配 `opacity`／`color-mix()`，不發明新 hex（CI 嚴格模式已有此規則，這裡是同一件事的視覺理由）

## 5. 反 AI slop 黑名單

以下是 AI 默認美學的簽名，出現即說明沒有在設計。每條附「為什麼」：

| 禁止 | 為什麼 |
|---|---|
| 紫色／藍紫漸變背景 | AI 訓練語料裡「科技感」的萬能公式，跟任何公司的品牌都無關 |
| emoji 當圖標（每個 bullet／標題配一個） | 「不夠專業就用 emoji 湊」；資訊沒有增加，噪音增加了 |
| 每個標題都配裝飾 icon | icon 只在**承載差異化資訊**時用（例如狀態、類型區分），裝飾性 icon 一律拿掉 |
| 圓角卡片＋左側彩色 border accent 的無限重複 | 2020-2024 模板時代的視覺噪音；一頁最多用在一種語義（例如僅警示框） |
| 編造 stats／quotes 當裝飾 | PM 文件的數字只能來自 context pack 或使用者輸入；沒有真數據就不放數字區塊，不要生「87% 用戶滿意」這種裝飾數據 |
| 均一的裝飾漸變、陰影堆疊 | 不攜帶資訊的視覺效果都是負資產 |
| SVG 手畫人物／場景插圖 | 畫出來永遠五官錯位；prototype 用灰色方塊＋文字標籤誠實佔位 |

**唯一合法例外**：`brand-tokens.css` 或 `brand-voice.md` 明確定義的樣式——品牌自己的簽名不是 slop。

**誠實佔位原則**：沒圖就留標注佔位的灰塊，沒數據就明講「待補」，一個誠實的 placeholder 比一個拙劣的假內容好十倍。

## 6. 資訊密度

- 每個元素要 earn its place：刪掉它版面會變差才留；填滿空間不是理由
- 密度要呼應內容性質：數據型段落（成功指標、排序表）可以密——用小字級、緊行高、表格化呈現撐起資訊量；敘事型段落（背景、動機）要疏——大字級、寬行高、短段落
- 一頁之內要有密度對比：全頁均一密度（不管均一地密還是均一地疏）都讀起來像沒有重點

## 7. 細節簽名：一處做到 120%

每份產出選**一個**細節做到超出預期，其他做到紮實即可——品味是在合適的地方精緻，不是均勻用力。例：

- report 的關鍵指標數字用大字級輕字重＋accent 色點綴
- 成功指標表的目標欄用視覺化的小進度標記
- prototype 完成畫面的一個有節制的完成狀態設計

一個明顯用心的細節，就能讓整份產出的觀感從「生成的」變成「設計的」。不要做五個 120%——那叫裝飾堆砌。

## 8. 自包含鐵律（比參考來源更嚴，不放鬆）

- **不外連任何資源**：CDN script、Google Fonts、外部圖片、icon library 一律禁止——企業內部文件會被離線開啟、內網轉發、郵件夾帶，外連資源等於在別人電腦上破版
- tokens 的 `:root` 變數整段複製進 `<style>`；需要 icon 用 inline SVG 或字符；需要圖表用 inline SVG 手繪（數據來自真實輸入）
- 單一 HTML 檔就是完整交付物，挪到任何地方都長一樣

## 9. 交付前 craft 自檢（可證偽，答不出＝沒過）

產出每份 HTML 後（在 CI 顏色自審之外）回答：

1. **「這份版面有哪一個視覺決定是從這次的內容長出來的？」**——指得出具體對應（例如「這次的核心是三案排序，所以排序表做成全頁視覺重心」）才算過；答不出來＝在套模板，回去重排
2. **「密度對比在哪？」**——指出頁面裡最密和最疏的兩個區塊；指不出來＝均一密度
3. **「accent 色出現在哪幾個位置、為什麼是那幾個？」**——數得出來（≤ 大約 5 處）且每處有理由才算紀律；「到處都有」＝沒有紀律
4. **「120% 的細節是哪一個？」**——說得出一個，且只有一個

這四題的答案不用寫進交付物，但驗證截圖肉眼審時要能對著畫面回答。
